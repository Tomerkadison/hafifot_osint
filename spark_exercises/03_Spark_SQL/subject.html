Spark SQL

לכתוב SQL רגיל על DataFrames — אותו מנוע, שתי שפות.

1אותו מנוע, שתי שפות

זו אחת התכונות הכי שימושיות ב-Spark: DataFrame ו-SQL הם אותו מנוע בדיוק. כל שאילתת SQL עוברת דרך אותו Catalyst optimizer כמו ה-API, ומחזירה DataFrame רגיל.

המשמעות: אתם יכולים לכתוב חלק מה-pipeline ב-SQL, חלק ב-API, ולעבור ביניהם באמצע — מה שנוח יותר לכל שלב.

2שלושת השלבים

  1. רושמים DataFrame כ-temp view: df.createOrReplaceTempView("orders")
  2. כותבים SQL: spark.sql("SELECT ... FROM orders")
  3. מקבלים DataFrame בחזרה — וממשיכים עם show, filter, או SQL נוסף.

ל-multi-line SQL השתמשו במחרוזת משולשת """...""".

3API מול SQL — אותה תוצאה

שתי הדרכים שקולות לחלוטין:

DataFrame API

  • orders.filter(F.col('quantity')>0)
  • .groupBy('channel')
  • .agg(F.sum('unit_price'))

Spark SQL

  • SELECT channel, sum(unit_price)
  • FROM orders WHERE quantity>0
  • GROUP BY channel
💡 גם F.expr("...") מאפשר לשתול קטע SQL בתוך ה-API — מושלם ל-CASE WHEN מסובך.

⚠️ createOrReplaceTempView חי רק כל עוד ה-SparkSession פתוח, ורק ב-session הנוכחי. ל-view שנראה מכל ה-sessions יש createGlobalTempView.
