structs, arrays ו-explode — כאן Spark עוקף בענק את pandas/Polars.

1מידע אמיתי הוא מקונן

אירועים, לוגים, ותגובות API כמעט תמיד מגיעים כ-JSON מקונן: אובייקט בתוך אובייקט, ומערכים בתוך שדות. ב-pandas/Polars זה כאב ראש — לרוב משטחים הכול או שומרים מחרוזות JSON.

Spark קורא JSON ישירות לטיפוסים מובנים: struct (אובייקט), array (מערך), ו-map (מילון). ה-schema נשמר, וה-Catalyst optimizer יודע לעבוד איתו.

2structs — גישה עם נקודה

שדה מקונן הוא פשוט parent.child:

  • events.select("device.os") — שדה בודד.
  • events.select("device.*") — לשטח את כל השדות.
  • F.col("geo.country") — בתוך filter או groupBy.

אפשר גם לבנות struct עם F.struct("a", "b").

3arrays — explode הוא הכוכב

מערך בתוך שורה (סל קניות, רשימת תגיות) זה נפוץ מאוד. הכלים:

  • F.size(arr) — אורך המערך.
  • F.explode(arr) — שורה לכל איבר (שורה אחת עם 3 פריטים → 3 שורות). זו הפעולה המרכזית.
  • F.posexplode(arr) — כמו explode, אבל גם המיקום.
  • F.array_contains, arr[0], F.explode_outer (שומר שורות עם מערך ריק/null).

אחרי explode עובדים על השורות כרגיל — groupBy, sum, join.

4מידע מקונן: pandas/Polars מול Spark

🐼 pandas / ⚡ Polars

  • JSON לרוב משוטח ידנית
  • מערכים בעמודה = object dtype מסורבל
  • json_normalize / list-cols
  • אין schema אכיפה

🔥 Spark

  • read.json שומר struct/array
  • device.os גישה ישירה
  • explode / from_json מובנים
  • schema מלא ונאכף
💡 ב-Foundry הרבה datasets הם בדיוק כאלה — אירועים מקוננים. explode ו-from_json הם לחם-חוק.
